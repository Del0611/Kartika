# Pendahuluan

Perangkat lunak dan programmer adalah dua elemen utama yang membentuk dunia teknologi informasi modern. Tanpa perangkat lunak, komputer tidak dapat berfungsi. Tanpa programmer, perangkat lunak tidak dapat dibuat. Materi ini membahas secara detail mengenai perangkat lunak serta peran seorang programmer.

---

# Perangkat Lunak (Software)

## 1. Definisi Perangkat Lunak

**Perangkat lunak (software)** adalah sekumpulan instruksi yang ditulis dalam bahasa pemrograman dan dijalankan oleh komputer untuk melakukan tugas tertentu. Software mengatur cara komputer bekerja dan berinteraksi dengan pengguna.

Berbeda dengan **perangkat keras (hardware)** yang berwujud fisik, perangkat lunak tidak berwujud namun sangat penting untuk menjalankan sistem komputer.

---

## 2. Kategori Perangkat Lunak

### 📌 A. Perangkat Lunak Sistem (System Software)

Perangkat lunak yang mengelola aktivitas dasar komputer, seperti:

* **Sistem operasi** → Windows, Linux, macOS
* **Driver perangkat keras**
* **Utility system** → antivirus, defrag, backup tools

### 📌 B. Perangkat Lunak Aplikasi (Application Software)

Software yang digunakan untuk membantu pekerjaan pengguna, seperti:

* Microsoft Office
* Browser (Chrome, Firefox)
* Editor gambar (Photoshop, GIMP)

### 📌 C. Perangkat Lunak Pengembangan (Programming Software)

Digunakan untuk membuat software lain:

* Compiler
* Code editor
* Debugger
* IDE (Visual Studio, PyCharm)

### 📌 D. Perangkat Lunak Berbasis Web

Berjalan melalui browser:

* Google Docs
* YouTube Studio
* Webmail

### 📌 E. Perangkat Lunak Mobile

Aplikasi untuk smartphone:

* WhatsApp
* Instagram
* Mobile banking

---

## 3. Siklus Hidup Perangkat Lunak (SDLC)

SDLC (Software Development Life Cycle) adalah proses yang digunakan untuk merancang, mengembangkan, dan memelihara software.

### Tahapan SDLC:

1. **Planning** – identifikasi kebutuhan
2. **Analysis** – analisis alur kerja dan solusi
3. **Design** – perancangan arsitektur dan tampilan
4. **Implementation** – penulisan kode program
5. **Testing** – memastikan software bebas error
6. **Deployment** – peluncuran software
7. **Maintenance** – pembaruan dan perbaikan

---

| Aplikasi       | Excel, Zoom              |

# Programmer

## 1. Definisi Programmer

**Programmer** adalah seseorang yang menulis, menguji, dan memelihara kode program untuk membuat sebuah perangkat lunak. Programmer menggunakan bahasa pemrograman untuk memberi instruksi kepada komputer.

---

## 2. Jenis-Jenis Programmer

### 🔧 A. Front-End Developer

Fokus pada tampilan dan interaksi pengguna.
Bahasa yang digunakan:

* HTML
* CSS
* JavaScript
* Framework: React, Vue, Angular

### 🔧 B. Back-End Developer

Mengurus server, database, dan logika aplikasi.
Bahasa:

* PHP
* Python
* Java
* Node.js
* Framework: Laravel, Django, Spring

### 🔧 C. Full-Stack Developer

Menguasai front-end dan back-end sekaligus.

### 🔧 D. Mobile Developer

Membuat aplikasi Android/iOS.
Bahasa:

* Kotlin
* Swift
* Flutter (Dart)

### 🔧 E. Game Developer

Membuat game menggunakan:

* Unity (C#)
* Unreal Engine (C++)

### 🔧 F. Data Scientist / Machine Learning Engineer

Membangun model AI.
Bahasa:

* Python
* R

### 🔧 G. DevOps Engineer

Mengintegrasikan development dan operations.
Tools:

* Docker
* Kubernetes
* CI/CD

---

## 3. Skill yang Dibutuhkan Programmer

### 🧠 Hard Skill

* Penguasaan bahasa pemrograman
* Algoritma dan struktur data
* Pemahaman database
* Version control (Git)
* Debugging
* Pengetahuan jaringan (untuk web)

### 🤝 Soft Skill

* Problem solving
* Critical thinking
* Kerja tim
* Manajemen waktu
* Komunikasi yang baik

---

## 4. Tools yang Digunakan Programmer

* **Code Editor**: VS Code, Notepad++, Vim
* **IDE**: IntelliJ, Visual Studio, Android Studio
* **Version Control**: Git, GitHub, GitLab
* **Database Tools**: MySQL Workbench, pgAdmin
* **Testing Tools**: Jest, PHPUnit, Selenium
* **DevOps Tools**: Docker, Jenkins

# Pengenalan Dunia Pengembangan Perangkat Lunak 🚀

Pengembangan perangkat lunak adalah proses membuat, merancang, menerapkan, dan memelihara program komputer untuk berbagai kebutuhan dan tujuan. Dunia pengembangan perangkat lunak sangat dinamis dan terus berkembang seiring kemajuan teknologi.

---

## Apa itu Pengembangan Perangkat Lunak? 💻

Pengembangan perangkat lunak melibatkan serangkaian aktivitas mulai dari:

- **Analisis kebutuhan** 🧐: Mengumpulkan dan memahami kebutuhan pengguna agar perangkat lunak dapat memenuhi tujuan yang diinginkan.
- **Desain sistem** 🛠️: Merancang struktur perangkat lunak dan komponen utamanya agar berjalan efisien dan mudah dikembangkan.
- **Pemrograman** ✍️: Menulis kode menggunakan bahasa pemrograman untuk mengimplementasikan desain.
- **Pengujian** ✅: Memastikan perangkat lunak berfungsi sesuai spesifikasi tanpa bug.
- **Pemeliharaan** 🔧: Memperbaiki masalah dan melakukan pembaruan agar perangkat lunak tetap relevan.

---

## Komponen Utama dalam Pengembangan Perangkat Lunak 🧩

| Komponen          | Penjelasan                                                                                               | Ikon   |
|-------------------|--------------------------------------------------------------------------------------------------------|--------|
| Analisis Kebutuhan| Memahami apa yang dibutuhkan pengguna dan mendokumentasikan fitur yang harus ada.                       | 🧐     |
| Desain Sistem     | Membuat blueprint aplikasi termasuk user interface dan pola arsitektur.                                | 🛠️    |
| Pemrograman       | Menulis kode menggunakan bahasa pemrograman tertentu untuk membangun aplikasi.                          | ✍️     |
| Pengujian         | Menjalankan berbagai tes untuk memastikan software bebas dari kesalahan.                               | ✅     |
| Pemeliharaan      | Melakukan update dan perbaikan setelah software dirilis agar tetap berjalan dengan baik.                | 🔧     |

---

## Bahasa Pemrograman Populer 🗣️

Bahasa pemrograman adalah alat utama dalam pengembangan perangkat lunak. Berikut beberapa bahasa yang sering digunakan:

- **Python** 🐍: Bahasa serbaguna dan mudah dipelajari, digunakan untuk web, AI, dan otomasi.
- **JavaScript** 🌐: Bahasa utama untuk pengembangan web interaktif di sisi klien.
- **Java** ☕: Cocok untuk aplikasi enterprise dan mobile (Android).
- **C#** 🎯: Banyak digunakan dalam pengembangan aplikasi Windows dan game.
- **Ruby** 💎: Terkenal untuk pengembangan web dengan framework Ruby on Rails.

---

## Metodologi Pengembangan 🧭

Metodologi pengembangan membantu tim bekerja secara terstruktur dan efisien:

- **Waterfall** 💧: Proses linear dan bertahap, cocok untuk proyek dengan kebutuhan stabil.
- **Agile** ⚡: Iteratif, mengutamakan kolaborasi dan respons cepat terhadap perubahan.
- **Scrum** 🏉: Framework Agile dengan tim kecil yang fokus pada sprint periode pendek.
- **DevOps** 🔄: Integrasi erat antara pengembangan dan operasi untuk mempercepat rilis produk.

# 📚 Jenis-Jenis Tipe Data + Penjelasannya

## 1. Tipe Data Dasar (Primitif)

👉 Tipe data paling sederhana, **tidak bisa dipecah lagi**

* **Integer (int)**
  Digunakan untuk menyimpan **bilangan bulat (tanpa koma)**
  Contoh: `5`, `-10`
  👉 Cocok untuk: jumlah orang, umur, jumlah barang

* **Float / Double**
  Digunakan untuk **bilangan desimal (pakai koma)**
  Contoh: `3.14`, `2.75`
  👉 Bedanya:

  * Float → lebih cepat tapi kurang akurat
  * Double → lebih akurat (lebih banyak angka di belakang koma)

* **Boolean (bool)**
  Hanya punya **2 nilai: benar atau salah**
  Contoh: `true`, `false`
  👉 Dipakai untuk: kondisi (if, perbandingan)

* **Character (char)**
  Menyimpan **1 karakter saja**
  Contoh: `'A'`, `'%'`
  👉 Beda dengan string (yang bisa banyak huruf)

* **Byte**
  Menyimpan angka kecil (0–255)
  👉 Hemat memori, sering dipakai untuk file atau data kecil

---

## 2. Tipe Data Gabungan (Composite)

👉 Tipe data yang **terdiri dari beberapa data lain**

* **String**
  Kumpulan karakter (teks)
  Contoh: `"Halo"`
  👉 Sebenarnya terdiri dari banyak char

* **Array (Larik)**
  Kumpulan data **dengan tipe yang sama** dalam satu urutan
  Contoh: `[1, 2, 3]`
  👉 Tiap data punya indeks (posisi)

* **Record / Struct**
  Gabungan beberapa data **dengan tipe berbeda**
  Contoh:
  Nama (string), Umur (int), Alamat (string)
  👉 Dipakai untuk menyimpan data yang saling berhubungan

* **Pointer**
  Menyimpan **alamat memori** dari variabel lain
  👉 Biasanya dipakai di bahasa seperti C/C++
  👉 Lebih ke konsep tingkat lanjut

---

## 3. Tipe Data Koleksi (ADT)

👉 Digunakan untuk **mengatur dan mengelola banyak data**

* **List**
  Seperti array tapi **ukurannya bisa berubah**
  👉 Bisa tambah/hapus data kapan saja

* **Map / Dictionary**
  Data dalam bentuk **pasangan kunci dan nilai**
  Contoh:
  `"nama": "Andi"`
  👉 Memudahkan pencarian data

* **Set**
  Kumpulan data yang **tidak boleh ada duplikat**
  Contoh: `{1, 2, 3}` (tidak bisa `{1,1,2}`)
  👉 Cocok untuk data unik

* **Stack (Tumpukan)**
  Menggunakan prinsip **LIFO (Last In, First Out)**
  👉 Yang terakhir masuk, keluar dulu
  Contoh: tumpukan piring

* **Queue (Antrean)**
  Menggunakan prinsip **FIFO (First In, First Out)**
  👉 Yang pertama masuk, keluar dulu
  Contoh: antrean di kasir

---

## 4. Tipe Data Khusus & Modern

👉 Digunakan untuk kebutuhan tertentu dalam program

* **Date & Time**
  Untuk menyimpan **tanggal dan waktu**
  👉 Contoh: jadwal, timestamp

* **Enum (Enumeration)**
  Kumpulan nilai tetap yang sudah ditentukan
  Contoh:
  `Warna = {MERAH, HIJAU, KUNING}`
  👉 Membatasi pilihan agar lebih aman

* **JSON / XML**
  Format data untuk **pertukaran informasi antar sistem**
  👉 Banyak dipakai di web & API

* **Null / Void**

  * **Null** → tidak ada nilai
  * **Void** → fungsi tidak mengembalikan nilai
    👉 Penting untuk menandai “kosong”

* **Any / Dynamic**
  Tipe data yang **bisa berubah-ubah**
  👉 Fleksibel, tapi harus hati-hati (bisa bikin error)
