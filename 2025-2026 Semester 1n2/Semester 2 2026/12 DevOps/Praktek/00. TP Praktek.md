
## **Materi Pembelajaran untuk Developer (DevOps Focus)**

### **1. Menggunakan Git untuk Version Control**
**Tujuan:** Siswa bisa menyimpan, melacak, dan berkolaborasi dalam pengembangan kode.
**Alat:** Git, GitHub/GitLab, VS Code.
**Langkah-langkah:**
1. **Install Git** dari [git-scm.com](https://git-scm.com/).
2. **Buat akun GitHub/GitLab**.
3. **Buat repository baru** (misal: `belajar-devops`).
4. **Di terminal:**
   ```bash
   git clone [link-repository]
   cd belajar-devops
   echo "# Halo DevOps" >> README.md
   git add .
   git commit -m "Pertama commit"
   git push origin main
   ```

---

### **2. Membuat Dockerfile untuk Aplikasi Sederhana**
**Tujuan:** Siswa bisa mengemas aplikasi ke dalam container.
**Alat:** Docker, text editor.
**Langkah-langkah:**
1. **Install Docker Desktop** dari [docker.com](https://www.docker.com/).
2. **Buat file `Dockerfile`:**
   ```dockerfile
   FROM nginx:alpine
   COPY index.html /usr/share/nginx/html
   ```
3. **Build image:**
   ```bash
   docker build -t web-sederhana .
   ```

---

### **3. Menggunakan CI/CD dengan GitHub Actions**
**Tujuan:** Siswa bisa mengotomatiskan pengujian dan build aplikasi.
**Alat:** GitHub, Docker Hub.
**Langkah-langkah:**
1. **Buat file `.github/workflows/ci.yml`:**
   ```yaml
   name: CI Pipeline
   on: [push]
   jobs:
     build:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v2
         - name: Build Docker Image
           run: docker build -t web-sederhana .
   ```

### **4. Menulis Unit Test untuk Aplikasi**
**Tujuan:** Siswa bisa menguji kode secara otomatis.
**Alat:** Python, pytest (atau PHPUnit untuk PHP).
**Langkah-langkah:**
1. **Buat file `test_kalkulator.py`:**
   ```python
   def test_tambah():
       assert 1 + 1 == 2
   ```
2. **Jalankan test:**
   ```bash
   pytest test_kalkulator.py
   ```

### **7. Menggunakan Kubernetes untuk Deploy Aplikasi**
**Tujuan:** Siswa bisa mendploy aplikasi ke cluster Kubernetes.
**Alat:** Minikube, kubectl.
**Langkah-langkah:**
1. **Install Minikube** dan jalankan cluster lokal.
2. **Deploy aplikasi:**
   ```bash
   kubectl create deployment hello-app --image=web-sederhana
   kubectl expose deployment hello-app --type=NodePort --port=80
   ```

## **Materi Pembelajaran untuk Operations (DevOps Focus)**

### **1. Mengelola Server dengan Ansible**
**Tujuan:** Siswa bisa mengatur server secara otomatis.
**Alat:** Ansible, VirtualBox.
**Langkah-langkah:**
1. **Install Ansible:**
   ```bash
   sudo apt update
   sudo apt install ansible
   ```
2. **Buat playbook** untuk menginstal Nginx:
   ```yaml
   - hosts: webservers
     tasks:
       - name: Install Nginx
         apt: name=nginx state=latest
   ```

---

### **2. Mengatur CI/CD Pipeline dengan Jenkins**
**Tujuan:** Siswa bisa mengotomatiskan deploy aplikasi.
**Alat:** Jenkins, Docker.
**Langkah-langkah:**
1. **Install Jenkins:**
   ```bash
   docker run -d -p 8080:8080 jenkins/jenkins
   ```
2. **Buat job baru** untuk build dan deploy aplikasi.

---

### **3. Mengamankan Server dengan Firewall dan SSL**
**Tujuan:** Siswa bisa mengamankan server dan aplikasi web.
**Alat:** UFW, Certbot.
**Langkah-langkah:**
1. **Aktifkan firewall:**
   ```bash
   sudo ufw allow 22
   sudo ufw allow 80
   sudo ufw allow 443
   sudo ufw enable
   ```
2. **Pasang SSL dengan Certbot:**
   ```bash
   sudo certbot --nginx -d namadomain.com
   ```

---

### **4. Membuat Backup dan Restore Database Otomatis**
**Tujuan:** Siswa bisa melindungi data aplikasi.
**Alat:** MySQL, cron.
**Langkah-langkah:**
1. **Buat script backup:**
   ```bash
   mysqldump -u root -p nama_database > backup.sql
   ```
2. **Jadwalkan backup otomatis** dengan cron.

### **7. Menggunakan Terraform untuk Infrastructure as Code**
**Tujuan:** Siswa bisa mengelola infrastruktur secara deklaratif.
**Alat:** Terraform, AWS/Azure/GCP (simulasi dengan LocalStack).
**Langkah-langkah:**
1. **Install Terraform** dari [terraform.io](https://www.terraform.io/).
2. **Buat file `main.tf`:**
   ```hcl
   resource "aws_instance" "web" {
     ami           = "ami-0c55b159cbfafe1f0"
     instance_type = "t2.micro"
   }
   ```

### **8. Mengatur Load Balancer untuk Aplikasi**
**Tujuan:** Siswa bisa mendistribusikan traffic ke beberapa server.
**Alat:** Nginx, Docker.
**Langkah-langkah:**
1. **Konfigurasi Nginx** sebagai load balancer:
   ```nginx
   upstream backend {
     server 192.168.1.10:80;
     server 192.168.1.11:80;
   }
   server {
     listen 80;
     location / {
       proxy_pass http://backend;
     }
   }
   ```

### **9. Mengelola User dan Izin Akses di Server**
**Tujuan:** Siswa bisa mengatur keamanan akses server.
**Alat:** Linux (Ubuntu/CentOS).
**Langkah-langkah:**
1. **Buat user baru:**
   ```bash
   sudo adduser devops
   ```
2. **Atur izin akses:**
   ```bash
   sudo usermod -aG sudo devops
   ```
