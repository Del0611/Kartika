
GitHub adalah **platform berbasis web** yang digunakan untuk **menyimpan, mengelola, dan berkolaborasi** dalam pengembangan perangkat lunak (software). GitHub menggunakan **Git** sebagai sistem version control (pengelolaan versi kode).

---

## **1. Apa Itu GitHub?**
- **GitHub** = Tempat menyimpan kode secara online, seperti Google Drive untuk developer.
- **Git** = Sistem yang melacak perubahan pada kode (siapa yang mengubah, kapan, dan apa yang diubah).
- **Fungsi Utama:**
  - Menyimpan kode secara aman.
  - Memudahkan kolaborasi tim.
  - Melacak sejarah perubahan kode.

---

## **2. Istilah Penting di GitHub**
| Istilah         | Penjelasan                                                                 |
|-----------------|----------------------------------------------------------------------------|
| **Repository**  | Folder tempat menyimpan semua file kode dan sejarah perubahan.          |
| **Commit**      | Catatan perubahan yang dilakukan pada kode.                              |
| **Branch**      | Salinan kode yang bisa diubah tanpa mengganggu kode utama.                |
| **Pull Request**| Permintaan untuk menggabungkan perubahan dari branch ke branch utama.     |
| **Fork**        | Menyalin repository orang lain ke akun GitHub sendiri.                    |
| **Clone**       | Mengunduh repository dari GitHub ke komputer lokal.                      |
| **Push**        | Mengunggah perubahan dari komputer lokal ke GitHub.                      |
| **Pull**        | Mengunduh perubahan terbaru dari GitHub ke komputer lokal.               |

---

## **3. Cara Menggunakan GitHub (Langkah demi Langkah)**

### **A. Membuat Akun GitHub**
1. Buka [github.com](https://github.com/).
2. Klik **Sign up** dan isi data diri.
3. Verifikasi email.

---

### **B. Membuat Repository Baru**
1. Klik tombol **+** di pojok kanan atas > **New repository**.
2. Isi **Repository name** (misal: `belajar-devops`).
3. Pilih **Public** (bisa dilihat semua orang) atau **Private** (hanya Anda/tim).
4. Klik **Create repository**.

---

### **C. Mengunggah Kode ke GitHub (Push)**
1. **Install Git** di komputer:
   - Download dari [git-scm.com](https://git-scm.com/).
   - Ikuti petunjuk instalasi.
2. **Buka terminal/cmd** dan jalankan perintah berikut:
   ```bash
   git config --global user.name "NamaAnda"
   git config --global user.email "email@anda.com"
   ```
3. **Clone repository** ke komputer:
   ```bash
   git clone https://github.com/username/namarepository.git
   cd namafile
   ```
4. **Tambahkan file kode** (misal: `index.html`).
5. **Simpan perubahan** ke GitHub:
   ```bash
   git add .
   git commit -m "Pesan Update"
   git push origin main
   ```

---

### **D. Mengunduh Perubahan dari GitHub (Pull)**
Jika ada perubahan di GitHub (misal: teman menambahkan kode), unduh perubahan terbaru:
```bash
git pull origin main
```

---

### **E. Membuat Branch Baru**
Branch digunakan untuk mengembangkan fitur baru tanpa mengganggu kode utama.
```bash
git branch fitur-login  # Membuat branch baru
git checkout fitur-login  # Pindah ke branch tersebut
```
Setelah selesai, gabungkan (merge) ke branch utama:
```bash
git checkout main
git merge fitur-login
git push origin main
```

---

### **F. Pull Request (PR)**
Jika bekerja dalam tim, gunakan **Pull Request** untuk menggabungkan kode:
1. **Push branch** ke GitHub.
2. Buka repository di GitHub > Klik **Pull Request** > **New Pull Request**.
3. Pilih branch yang ingin digabungkan > Klik **Create Pull Request**.
4. Tim akan mereview kode sebelum digabungkan.

---

### **G. Fork Repository**
Jika ingin berkontribusi pada repository orang lain:
1. Buka repository target > Klik **Fork** (pojok kanan atas).
2. Repository akan tersalin ke akun GitHub Anda.
3. **Clone repository fork** ke komputer, lakukan perubahan, lalu ajukan **Pull Request** ke repository asli.

---

## **4. Contoh Penggunaan GitHub untuk Tugas Sekolah**
**Skenario:** Kelompok membuat website sekolah.
1. **Satu orang** membuat repository di GitHub (misal: `website-smk`).
2. **Semua anggota** melakukan **fork** repository tersebut.
3. Setiap anggota **clone repository fork** ke komputer mereka.
4. Setiap anggota bekerja di **branch sendiri** (misal: `fitur-berita`, `fitur-galeri`).
5. Setelah selesai, mereka **push** ke branch masing-masing dan ajukan **Pull Request** ke repository utama.
6. **Ketua kelompok** mereview dan **merge** Pull Request.

---

## **5. Tips Menggunakan GitHub**
- **Commit sering**: Simpan perubahan kecil-kecil agar mudah dilacak.
- **Gunakan pesan commit yang jelas**: Misal: "Menambahkan halaman kontak" (bukan "fix bug").
- **Gunakan `.gitignore`**: Untuk mengecualikan file yang tidak perlu diunggah (misal: file log, cache).
- **Pelajari Markdown**: Untuk membuat `README.md` yang rapi (penjelasan tentang repository).

---

## **6. Kesimpulan**
- GitHub = **Google Drive untuk developer** + **alat kolaborasi**.
- Git = **Sistem pelacak perubahan** pada kode.
- **Pull > Commit > Push** = Siklus dasar penggunaan GitHub.
