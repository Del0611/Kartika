# 📂 Modul: Dasar Basis Data (Database)

## 1. Pengertian Basis Data
**Basis Data** (Database) adalah kumpulan data yang terorganisir secara sistematis sehingga dapat disimpan, dikelola, dan diakses dengan cepat.

> **Analogi:** Jika sebuah folder di komputer adalah lemari, maka database adalah sistem pengarsipan profesional di dalamnya yang memungkinkan kita menemukan satu lembar kertas di antara jutaan kertas lainnya hanya dalam hitungan detik.

---

## 2. Mengapa Perlu Database?
Tanpa database, aplikasi web yang kita buat akan bersifat **statis** (datanya hilang saat browser ditutup). Database memberikan kemampuan:
* **Persistence:** Data tersimpan secara permanen.
* **Scalability:** Mampu menampung ribuan hingga jutaan data.
* **Security:** Data dapat dikunci dengan hak akses tertentu.
* **Efficiency:** Mempercepat pencarian dan pengolahan informasi.

---

## 3. Komponen Relational Database (RDBMS)
Hampir seluruh aplikasi web di kelas 11 RPL akan menggunakan **RDBMS**. Data di dalamnya disimpan dalam bentuk **Tabel**.

### A. Struktur Tabel
| Istilah | Penjelasan | Contoh |
| :--- | :--- | :--- |
| **Table** | Kumpulan data dalam tema tertentu. | Tabel `Siswa` |
| **Field (Column)** | Kategori informasi yang disimpan. | `Nama`, `NIS`, `Alamat` |
| **Record (Row)** | Satu baris data lengkap. | `101, Budi, Jakarta` |
| **Primary Key** | Identitas unik untuk tiap record. | `NIS` (Tidak boleh ada yang sama) |

---

## 4. Mengenal DBMS
**DBMS** (*Database Management System*) adalah perangkat lunak yang digunakan untuk mengelola database.

* **MySQL / MariaDB:** Paling populer di SMK karena gratis dan mudah (Bagian dari XAMPP/Laragon).
* **PostgreSQL:** Standar industri untuk data yang lebih kompleks.
* **SQLite:** Database berbentuk file tunggal, cocok untuk aplikasi kecil atau mobile.

---

## 5. Bahasa SQL (Structured Query Language)
SQL adalah bahasa standar yang digunakan untuk berkomunikasi dengan DBMS. Perintahnya dibagi menjadi tiga kelompok besar:

### 1. Data Definition Language (DDL)
Digunakan untuk mendefinisikan struktur database.
* `CREATE DATABASE`: Membuat database baru.
* `CREATE TABLE`: Membuat tabel baru.
* `DROP`: Menghapus tabel atau database.

### 2. Data Manipulation Language (DML)
Digunakan untuk mengolah isi data (Wajib dikuasai Programmer).
* **C**reate : `INSERT INTO ...` (Menambah data)
* **R**ead : `SELECT ... FROM ...` (Membaca data)
* **U**pdate : `UPDATE ... SET ...` (Mengubah data)
* **D**elete : `DELETE FROM ...` (Menghapus data)

### 3. Data Control Language (DCL)
Digunakan untuk mengatur hak akses (Admin).
* `GRANT`: Memberikan izin akses.
* `REVOKE`: Mencabut izin akses.

---

## 6. Alur Kerja Web Dinamis dengan Database
1.  **User** mengisi form di halaman web (Client).
2.  **Browser** mengirim data ke **Web Server**.
3.  **Server-Side Script** (PHP/Node.js) menerima data dan memprosesnya.
4.  **Server** mengirim perintah **SQL** ke **Database**.
5.  **Database** menyimpan data dan memberi konfirmasi balik ke Server.

---

## 7. Latihan Praktik (Konsep Dasar)
Bayangkan kamu diminta membuat database untuk **Sistem Perpustakaan**.

**Tugas Mandiri:**
1. Tentukan apa saja tabel yang dibutuhkan (Contoh: Tabel Buku, Tabel Anggota).
2. Tentukan apa saja kolom (field) untuk masing-masing tabel.
3. Tentukan kolom mana yang akan menjadi **Primary Key**.

---

Tipe data adalah aturan yang diterapkan pada setiap kolom (field) dalam tabel untuk menentukan jenis data apa yang boleh disimpan di sana. Memilih tipe data yang tepat sangat penting untuk efisiensi penyimpanan dan kecepatan pencarian data.

Berikut adalah materi lengkap mengenai tipe data pada basis data (khususnya MySQL/MariaDB) dalam format Markdown.

# 📊 Modul: Tipe Data pada Basis Data

Dalam basis data, kita tidak bisa asal memasukkan data. Setiap kolom harus memiliki **Tipe Data** yang spesifik agar komputer tahu cara memprosesnya (misal: angka bisa dijumlahkan, tapi teks tidak).

---

## 1. Tipe Data Angka (Numeric)
Digunakan untuk data berupa angka, baik angka bulat maupun pecahan.

| Tipe Data | Penjelasan | Penggunaan |
| :--- | :--- | :--- |
| **INT / INTEGER** | Angka bulat (tanpa koma). | Stok barang, Jumlah siswa, ID. |
| **TINYINT** | Angka bulat sangat kecil (0-255). | Status (0 atau 1), Umur. |
| **DECIMAL / FLOAT** | Angka pecahan (dengan koma). | Harga barang, Nilai IPK, Berat badan. |

> **Tips:** Gunakan `INT` untuk data yang akan dihitung (aritmatika). Jika angka tersebut tidak akan dihitung (seperti Nomor HP atau NISN), lebih baik gunakan tipe data teks.

---

## 2. Tipe Data Teks (String)
Digunakan untuk menyimpan karakter, kata, kalimat, atau angka yang tidak dihitung.

| Tipe Data | Penjelasan | Penggunaan |
| :--- | :--- | :--- |
| **CHAR(n)** | Teks dengan panjang tetap. | Kode produk (misal: "BRG01"). |
| **VARCHAR(n)** | Teks dengan panjang berubah-ubah (lebih hemat). | Nama, Alamat, Email, Password. |
| **TEXT** | Teks yang sangat panjang. | Artikel, Deskripsi produk, Komentar. |



---

## 3. Tipe Data Tanggal & Waktu (Date & Time)
Penting untuk mencatat kapan sebuah data dibuat atau kapan suatu kejadian berlangsung.

| Tipe Data | Format | Penggunaan |
| :--- | :--- | :--- |
| **DATE** | `YYYY-MM-DD` | Tanggal lahir, Tanggal pinjam. |
| **TIME** | `HH:MM:SS` | Jam masuk, Durasi pengerjaan. |
| **DATETIME** | `YYYY-MM-DD HH:MM:SS` | Waktu transaksi (lengkap). |
| **YEAR** | `YYYY` | Tahun terbit buku, Tahun angkatan. |

---

## 4. Tipe Data Lainnya
* **ENUM:** Digunakan untuk pilihan yang sudah pasti (Dropdown).
  * *Contoh:* `ENUM('Laki-laki', 'Perempuan')` atau `ENUM('S', 'M', 'L', 'XL')`.
* **BOOLEAN:** Hanya berisi dua nilai: `TRUE` atau `FALSE` (biasanya diwakili angka 1 atau 0).
* **BLOB:** Digunakan untuk menyimpan data biner seperti gambar atau file (namun biasanya programmer lebih suka menyimpan "path/alamat file"-nya saja dalam bentuk VARCHAR).

---

## 5. Mengapa Pemilihan Tipe Data itu Krusial?

1. **Efisiensi Ruang Simpan:** Menggunakan `BIGINT` untuk kolom yang hanya berisi angka 1-10 adalah pemborosan memori.
2. **Validasi Data:** Jika kolom bertipe `INT` dipaksa diisi huruf, database akan menolak (error). Ini menjaga data tetap bersih.
3. **Kecepatan Query:** Database jauh lebih cepat mencari data dalam format angka (`INT`) dibandingkan format teks (`VARCHAR`).

---

## 💡 Contoh Kasus: Tabel Siswa
Jika kita merancang tabel `siswa`, berikut adalah rancangan tipe datanya:

| Nama Kolom | Tipe Data | Alasan |
| :--- | :--- | :--- |
| `id_siswa` | **INT (Primary Key)** | Berupa angka unik untuk identitas. |
| `nisn` | **VARCHAR(10)** | Meski angka, NISN tidak untuk dijumlahkan. |
| `nama_lengkap`| **VARCHAR(100)** | Nama orang panjangnya bervariasi. |
| `jenis_kelamin`| **ENUM('L', 'P')** | Pilihannya sudah pasti hanya dua. |
| `tanggal_lahir`| **DATE** | Untuk menyimpan informasi tanggal. |
| `alamat` | **TEXT** | Alamat bisa sangat panjang. |
