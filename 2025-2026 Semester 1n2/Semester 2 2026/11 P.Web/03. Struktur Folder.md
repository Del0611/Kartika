## Bagian 1: Struktur Folder Proyek Web

Saat membuat website, Anda akan memiliki banyak file (HTML, CSS, JavaScript, gambar, PHP, dll). Jika semua file ditaruh di satu tempat tanpa diatur, proyek Anda akan menjadi berantakan (*spaghetti code*) dan sulit dikembangkan.

### Standar Struktur Folder yang Baik

Meskipun tidak ada aturan baku yang mutlak, industri web development umumnya menggunakan struktur folder yang rapi seperti ini:

```text
📁 nama-proyek-anda/
│
├── 📁 assets/                 └── (Tempat menyimpan file pendukung static)
│   ├── 📁 css/                └── (File gaya tampilan: style.css)
│   ├── 📁 js/                 └── (File logika interaktif: script.js)
│   └── 📁 images/             └── (Foto, logo, dan ikon: logo.png)
│
├── 📁 includes/               └── (Komponen web yang dipakai berulang kali)
│   ├── header.php
│   └── footer.php
│
├── index.php / index.html     └── (Halaman utama/gerbang masuk website)
├── tentang.php / tentang.html └── (Halaman pendukung)
└── kontak.php / kontak.html   └── (Halaman pendukung)

```

### 3 Aturan Penting Struktur Folder:

1. **Gunakan File `index` sebagai Gerbang:** Web server (seperti Apache di XAMPP/Laragon) secara otomatis akan mencari file bernama `index.html` atau `index.php` untuk ditampilkan pertama kali saat folder proyek diakses.
2. **Gunakan Huruf Kecil (Lowercase):** Sistem operasi server seperti Linux sangat sensitif terhadap huruf besar-kecil (*case-sensitive*). Folder `Assets` dan `assets` dianggap berbeda. Selalu gunakan huruf kecil untuk menghindari error saat website di-online-kan.
3. **Pisahkan Kode dan Aset:** Jangan pernah mencampur file gambar atau CSS langsung di folder utama (root). Masukkan ke dalam folder `assets`.

## Bagian 2: Protokol HTTP (Hypertext Transfer Protocol)

Setelah folder web Anda rapi di localhost, bagaimana caranya browser (seperti Chrome) bisa menampilkan halaman web tersebut? Jawabannya adalah melalui **HTTP**.

> **HTTP adalah** bahasa komunikasi atau protokol standar yang digunakan antara **Client** (Browser Anda) dan **Server** (XAMPP/Laragon atau komputer hosting) untuk saling bertukar data.

### Cara Kerja HTTP (Request & Response)

Proses komunikasi HTTP selalu berpasangan: Client meminta, Server menjawab.

```
[ CLIENT / BROWSER ]   ---- HTTP Request (Minta file) ---->   [ SERVER (XAMPP/Hosting) ]
[ CLIENT / BROWSER ]   <--- HTTP Response (Kirim file) ----   [ SERVER (XAMPP/Hosting) ]

```

1. **1. HTTP Request (Permintaan):** Client Side.
Saat Anda mengetik `localhost/toko-ku/index.html` lalu menekan Enter, browser Anda mengirimkan **HTTP Request** ke server lokal. Pesan ini berisi metode yang digunakan (misal: `GET`) dan file apa yang diminta.


2. **2. Pemrosesan oleh Server:** Server Side.
Web Server (Apache/Nginx) menerima permintaan tersebut, mencari file `index.html` di dalam folder `htdocs` atau `www`, dan menyiapkannya.


3. **3. HTTP Response (Tanggapan):** Client Side.
Server mengirimkan **HTTP Response** kembali ke browser. Respons ini membawa dokumen yang diminta beserta kode status untuk memberi tahu browser apakah prosesnya sukses atau gagal. Browser kemudian membaca kode HTML/CSS tersebut dan menampilkannya secara visual ke layar Anda.


### HTTP Status Code (Kode Status) yang Wajib Diketahui

Saat server memberikan respons, mereka selalu menyertakan kode angka 3 digit yang menunjukkan status dari permintaan tersebut:

* **`200 OK`** : Permintaan sukses! File ditemukan dan berhasil dikirim ke browser.
* **`301 Moved Permanently`** : Redirect. Halaman web dipindahkan ke URL yang baru.
* **`404 Not Found`** : Error dari sisi client. File yang diminta tidak ada di server (sering terjadi karena salah ketik nama file/folder).
* **`500 Internal Server Error`** : Error dari sisi server. Biasanya terjadi karena ada kesalahan penulisan kode PHP atau konfigurasi server yang rusak.

---

### Perbedaan HTTP vs HTTPS

Anda pasti sering melihat ikon gembok di browser saat berselancar di internet. Itu tandanya website tersebut menggunakan **HTTPS (HTTP Secure)**.

* **HTTP:** Data dikirim dalam bentuk teks biasa tanpa enkripsi. Jika ada peretas (*hacker*) di jaringan Wifi yang sama, mereka bisa mengintip data/password yang Anda masukkan.
* **HTTPS:** Data diacak (dienkripsi) menggunakan sertifikat SSL/TLS sebelum dikirim. Sangat aman untuk transaksi dan pengiriman data sensitif. Di lingkungan localhost, kita biasanya cukup memakai HTTP biasa.
